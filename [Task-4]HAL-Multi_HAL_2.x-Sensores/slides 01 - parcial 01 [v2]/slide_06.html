Drivers USB Serial Ativados

Habilitação de drivers essenciais no Kernel da Motorola (sm8250) para suporte a conversores serial-USB comuns em placas IoT.

CP210x CH341 FTDI CDC_ACM

Permissões de Dispositivo

Configuração do daemon ueventd para garantir que o nó /dev/ttyUSB* seja propriedade do usuário system, prevenindo captura pelo Bluetooth stack.

Políticas SELinux (Vendor)

Definição de regras estritas no contexto vendor para permitir que a HAL (hal_sensors_default) acesse os dispositivos seriais com operações específicas (ioctl, read, write).

nio-default.config

# --- Drivers USB Serial para Estação Ar (ESP32) ---

CONFIG_USB_SERIAL=y

CONFIG_USB_SERIAL_CP210X=y # ESP32 DevKit

CONFIG_USB_SERIAL_CH341=y # Clones Chineses

CONFIG_USB_SERIAL_FTDI_SIO=y # Industrial

CONFIG_USB_ACM=y # USB Nativo S2/S3

ueventd.qcom.rc

# --- Estação de Qualidade do Ar (ESP32) ---

# Garante acesso HAL (system) e previne BT

/dev/ttyUSB* 0660 system system
/dev/ttyACM* 0660 system system
sepolicy/vendor/airquality.te

# --- Política SELinux para AirQuality HAL ---

allow hal_sensors_default device:dir r_dir_perms;

 

# Permissão IOCTL essencial para BaudRate

allow hal_sensors_default usb_serial_device:chr_file {

open read write ioctl getattr lock

};
